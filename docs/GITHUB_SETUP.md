# GitHub setup guide

This guide is for users who normally only download files from GitHub and have never published a project.

## Create the repository

1. Log in to GitHub.
2. Click the `+` button in the top-right corner.
3. Select `New repository`.
4. Repository name:

```text
hibrew-h10-plus-pressure-control
```

5. Description:

```text
Arduino Nano pressure-control mod for the HiBREW H10A Plus espresso machine.
```

6. Set visibility to `Public`.
7. Add the MIT license if you want GitHub to create it, or upload the included `LICENSE` file.
8. Create the repository.

## Upload this project from the browser

1. Open the new repository.
2. Click `Add file`.
3. Click `Upload files`.
4. Drag the contents of this project folder into the upload area.
5. Commit message:

```text
Initial project release
```

6. Click `Commit changes`.

## Add repository topics

Open the repository page, click the gear icon near `About`, and add topics such as:

```text
arduino espresso coffee pressure-control hibrew h10a-plus vibration-pump ulka triac
```

## Recommended first release

After uploading the files:

1. Click `Releases`.
2. Click `Draft a new release`.
3. Tag version:

```text
v0.2.0
```

4. Release title:

```text
Working v0.2 firmware and documentation
```

5. Release notes:

```text
Initial public release with the tested Arduino Nano firmware, documentation, wiring notes, safety warnings, and diagrams.
```

## GitHub Pages

GitHub Pages is optional. The repository README already works as a project homepage.

Enable GitHub Pages only if you want a separate website later.
